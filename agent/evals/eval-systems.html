Bedrock Agent Eval Systems

New directory split for evals. This is a scaffold only; the legacy runner still lives at evals/run.mjs.

Change: stop treating evals as one giant suite with flags. Split by what is being tested: prompt behavior, memory quality, burst/session handling, and deterministic tools/PMS adapters.

Systems

1. Agent Prompt / Tool Calls

evals/agent-prompt/

  • Checks tool choice and tool args.
  • Checks PM, tenant, and vendor message bodies inside tool calls.
  • Checks context/lane separation and status updates.
  • Assumes memory/context has already been provided.

2. Memory

evals/memory/

  • Checks retrieval quality, observations, and beliefs.
  • Owns conditional/procedural memory once implemented.
  • Existing memory-graph gold files belong here conceptually.

3. Burst / Session

evals/burst-session/

  • Checks how raw inbound messages become turns.
  • Owns typo-then-correction and final-intent-wins cases.
  • Checks queued turns and revalidation before action.

4. Tool / PMS Adapter

evals/tool-pms-adapter/

  • Checks deterministic tool behavior below the model.
  • Checks AppFolio draft/send/update correctness.
  • Checks failure surfaces and side effects.
